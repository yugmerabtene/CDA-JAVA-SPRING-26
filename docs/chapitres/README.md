# Parcours par chapitres

Ce dossier contient le vrai cours du projet.

Le principe retenu est simple:
- un chapitre = une brique de l'application
- les chapitres suivent l'ordre chronologique de construction
- le code est presente sous forme de fichiers complets
- a la fin du parcours, on retombe sur l'application finale du depot

Ordre de lecture:

1. `../SYLLABUS.md`
2. `00-prerequis-et-socle.md`
3. `01-configuration-et-flyway.md`
4. `02-modele-jpa-et-enum-roles.md`
5. `03-repository-et-dao.md`
6. `04-dto-mapper-et-services.md`
7. `05-mvc-thymeleaf-bootstrap.md`
8. `06-spring-security.md`
9. `07-cache-redis.md`
10. `08-erreurs-metier.md`
11. `09-dockerisation.md`
12. `10-tests-integration.md`

Ce que tu trouveras dans chaque chapitre:
- l'objectif de la brique
- l'ordre exact d'implementation
- les fichiers complets a creer ou mettre a jour
- la commande de validation
- le resultat attendu avant de passer au chapitre suivant
